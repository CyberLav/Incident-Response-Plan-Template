# Annex C — SMESC Order Template

Used at [Containment, Step 13](../03-containment.md#36-step-13--prepare-and-issue-orders)
to issue a formal, unambiguous order to the technical team executing containment or
eradication. This is the single highest-leverage artifact the battle-procedure layer adds
to the plan — a structured order means the person executing knows exactly what to do, why,
and who to report to, without a scramble of chat messages.

```
INCIDENT ORDER — [Incident ID]
Issued by: [IC name/role]      Time: [timestamp]

SITUATION
  Friendly forces:   [systems/teams currently engaged]
  Threat:            [what we're dealing with, confidence level]

MISSION
  [One sentence: who does what, by when, to what standard]

EXECUTION
  Commander's intent: [the "why" — what success looks like]
  Concept of operations: [the COA selected, in sequence]
  Tasks by team/individual:
    - [Team/person]: [specific task] [deadline]
    - [Team/person]: [specific task] [deadline]
  Coordinating instructions: [dependencies, sequencing, do-not-do items]

SERVICE SUPPORT
  Tools/access required: [ ]
  Escalation path if blocked: [ ]

COMMAND & SIGNAL
  IC: [name]        Backup IC: [name]
  Reporting cadence: [e.g., SITREP every 30 min]
  Reporting channel: [out-of-band comms link]
```

> This template is also being built as an interactive fill-in tool on cyberlav.io — this
> file is the canonical schema it's built from.
